# Week 5 — Agentic AI Fundamentals

## Goal

Understand how AI agents **reason, plan, use tools, and execute tasks**.
This week introduces the foundations of **Agentic AI systems**.

The goal is to move from simple AI responses to **autonomous systems that can complete tasks**.

---

# What is Agentic AI?

Simple AI systems:

```
User → LLM → Answer
```

Agent systems:

```
User
 ↓
LLM reasoning
 ↓
Choose tool
 ↓
Execute tool
 ↓
Return result
```

Agent systems can **plan and execute multi-step tasks**.

---

# Core Components of an AI Agent

Every AI agent has **four basic abilities**.

```
1. Reasoning
2. Planning
3. Tool usage
4. Execution
```

---

## 1. Reasoning

The model **analyzes the request** and decides what the problem is.

Example:

```
User: Calculate 45 * 12
```

Agent reasoning:

```
This is a math problem → use calculator tool
```

---

## 2. Planning

The agent decides **which steps are needed to complete a task**.

Example request:

```
User: Generate Playwright test for login page
```

Plan:

```
Step1 Understand requirement
Step2 Search documentation
Step3 Generate automation code
```

---

## 3. Tool Usage

Agents use external tools to complete tasks.

Examples of tools:

```
Calculator
Vector database search
Browser automation
File reader
API calls
```

Example tools for a QA automation agent:

```
Jira reader
Testcase generator
Playwright runner
Failure analyzer
Report generator
```

---

## 4. Execution

The agent executes tasks sequentially.

Example workflow:

```
User request
↓
Agent reasoning
↓
Tool execution
↓
Final result
```

# Three Core Agent Patterns

Most AI agents follow **three common patterns**.

---

## Pattern 1 — Tool Agent

The LLM selects a tool.

Example:

```
User: What browsers does Playwright support?
```

Agent:

```
Use documentation search tool
```

Flow:

```
User
↓
Agent
↓
Tool selection
↓
Execute tool
↓
Answer
```

---

## Pattern 2 — RAG Agent

The agent retrieves knowledge before answering.

Architecture:

```
User
↓
Agent
↓
Vector search
↓
Context retrieved
↓
LLM generates answer
```

Technologies used:

* Chroma vector database
* Ollama LLM

---

## Pattern 3 — Multi-Step Agent

The agent performs **multiple steps sequentially**.

Example:

```
User: Generate Playwright test for login
```

Agent plan:

```
Step1 Generate testcases
Step2 Generate automation code
Step3 Validate script
Step4 Return result
```

Flow:

```
User
↓
Planner
↓
Step execution
↓
Final output
```

---

# The 5 Building Blocks of Production AI Agents

Production systems use **five building blocks**.

```
1. LLM Brain
2. Memory
3. Tools
4. Planner
5. Executor
```

---

## 1. LLM Brain (Reasoning Engine)

The LLM acts as the **brain of the agent**.

Examples:

* Ollama models like `phi3` or `mistral`
* GPT-style models

Responsibilities:

```
Understand request
Reason about task
Choose tools
Generate responses
```

Example reasoning:

```
User: Generate Playwright test for login
```

Brain decides:

```
Automation task → generate testcases → write code
```

---

## 2. Memory

Agents store information for context.

### Short-Term Memory

Conversation history.

Example:

```
User: Generate login test
User: Add invalid login case
```

Agent remembers the previous message.

---

### Long-Term Memory

Stored knowledge in vector databases such as **Chroma**.

Used for:

```
Company documentation
Automation guidelines
Test frameworks
Internal knowledge
```

---

## 3. Tools

Tools allow agents to **interact with the outside world**.

Examples:

```
Calculator
File reader
Vector search
Browser automation
API calls
```

Future tools for QA automation agent:

```
generate_testcases()
generate_automation()
run_playwright_tests()
analyze_failures()
generate_report()
```

---

## 4. Planner

The planner determines **task steps**.

Example request:

```
Create automation for login feature
```

Planner output:

```
Step1 Generate testcases
Step2 Generate automation code
Step3 Run tests
Step4 Analyze failures
```

Planner = **task decomposition**.

---

## 5. Executor

Executor runs the steps generated by the planner.

Example:

```
Step1 → run testcase generator
Step2 → run automation generator
Step3 → run tests
Step4 → analyze results
```
